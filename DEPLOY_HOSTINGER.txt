INSTRUÇÕES DE DEPLOY PARA HOSTINGER
=====================================

OPÇÃO A: DEPLOY AUTOMÁTICO VIA GIT (RECOMENDADO)
-------------------------------------------------
1. No painel Hostinger, vá em Advanced → Git
2. Clique em "Create new repository"
3. Configure:
   - Repository URL: https://github.com/seu-usuario/apafut.git
   - Branch: main
   - Deployment path: /public_html
4. Adicione sua chave SSH ou Personal Access Token
5. Clique em "Pull" para fazer o primeiro deploy
6. A cada push no GitHub, o deploy será automático

OPÇÃO B: GITHUB ACTIONS COM FTP
--------------------------------
1. No GitHub, vá em Settings → Secrets → Actions
2. Adicione os secrets:
   - FTP_USERNAME: seu usuário FTP da Hostinger
   - FTP_PASSWORD: sua senha FTP da Hostinger
3. Faça push do arquivo .github/workflows/deploy.yml
4. A cada push na branch main, o deploy será automático

OPÇÃO C: DEPLOY MANUAL (TRADICIONAL)
-------------------------------------

1. PREPARAÇÃO DO BANCO DE DADOS
   - No painel da Hostinger, crie um banco de dados MySQL
   - Anote: nome do banco, usuário, senha e host
   - Importe o arquivo: sql/database.sql
   - Execute os demais arquivos SQL na ordem:
     * sql/create_analytics_table.sql
     * sql/create_assinaturas_table.sql
     * sql/add_scheduling_fields.sql
     * sql/performance_indexes.sql
     * sql/remove_pendente_status.sql

2. CONFIGURAÇÃO DO BANCO
   - Edite o arquivo: config/db.php
   - Atualize as credenciais do banco de dados da Hostinger:
     * DB_HOST (geralmente localhost)
     * DB_NAME (u754804453_apafut)
     * DB_USER (usuário do banco)
     * DB_PASS (senha do banco)

3. CONFIGURAÇÃO DO .HTACCESS
   - Edite o arquivo: .htaccess
   - Na linha 14, substitua "seudominio.com.br" pelo seu domínio real
   - Exemplo: RewriteCond %{HTTP_REFERER} !^https?://(www\.)?apafut\.com\.br [NC]

4. UPLOAD DOS ARQUIVOS
   - Conecte via FTP ou File Manager da Hostinger
   - Faça upload de todos os arquivos para public_html/
   - NÃO envie os arquivos .md (já foram removidos)
   - NÃO envie a pasta .git

5. PERMISSÕES DE PASTAS
   - cache/ → 755 (permissão de escrita)
   - logs/ → 755 (permissão de escrita)
   - assets/images/jogadores/ → 755
   - assets/images/noticias/ → 755
   - assets/videos/ → 755

6. TESTE O SITE
   - Acesse: https://seudominio.com.br
   - Teste o login admin: https://seudominio.com.br/admin
   - Verifique se todas as páginas carregam corretamente

7. CONFIGURAÇÕES FINAIS
   - Configure SSL/HTTPS no painel da Hostinger
   - Verifique se o .htaccess está forçando HTTPS
   - Teste o formulário de checkout
   - Teste o sistema de upload de imagens

8. MANUTENÇÃO
   - Logs de erro: /logs/
   - Cache: /cache/ (limpar se necessário)
   - Backup regular do banco de dados

IMPORTANTE:
- Todos os arquivos markdown foram removidos
- O .htaccess está configurado para produção com HTTPS
- Certifique-se de atualizar o domínio no .htaccess
